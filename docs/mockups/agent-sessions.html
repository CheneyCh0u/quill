3 · 隔离模型与数据落点（实现约定）

① 会话归属 scope

会话挂在现有 scope（桌面：workspace / file；web：当前云端工作区）之下。 切 scope 看到的是各自的会话列表，互不可见——延续现有上下文按 scope 隔离的语义，往下再切一层。

② 桌面端存储

现状 ~/.quill/contexts/<scopeKey>.json（单文件 = 单对话） → contexts/<scopeKey>/<sessionId>.json ＋ 目录内 index.json（标题/时间/轮数/活动会话）。 启动时发现旧单文件 → 自动迁移成该 scope 的第一个会话，老对话不丢。

③ web 端存储

现状 localStorage 单 key → quill:agent:<workspaceId>:<sessionId> ＋ 会话索引 key。同样自动迁移旧 key 为第一个会话。 （跨端会话同步不在本期：桌面 agent 本地跑、web agent 服务端跑，历史本就不互通。）

  • 01 新建会话 = 旧「清空」的替代：当前对话原样留在列表里，新会话空白起步。运行中（agent 正在跑）禁用切换/新建/删除，先取消或等结束。
  • 02 删除会话：行内二次确认；删除的是磁盘/localStorage 里的上下文，不可恢复。删当前 → 切最近；删到空 → 自动建新空会话。
  • 03 自动压缩、token 预算、审批流等现有机制不变，全部作用于「当前会话」。导出 markdown 导出当前会话。
  • 04 web 端切云端工作区时，面板加载该工作区的活动会话（之前是 reset；现在变成「回到那个工作区的上次会话」，体验更顺）。
