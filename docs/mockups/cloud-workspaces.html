3 · Agent 权限域：钉死在当前工作区

现状

web 端 agent run 时 server 强制 scope.root = vault 根， 即 agent 可读写整个 vault 任意文件。

改为

run 消息带 workspaceId，server 对照注册表校验后 scope.root = vault/<工作区目录>。 非法 / 不存在的 id 直接拒绝。agent 工具的路径守卫保证不越界。

web 端 AI 面板顶部加一行权限提示：agent 可访问：/quill，切工作区时对话按现有 scope 机制重置。

4 · 数据模型与迁移（实现约定）

① 注册表统一升级

/api/sync/spaces → /api/workspaces（路由改名，桌面端引擎同步更新）。 条目即工作区：{id, name, remotePath, default?, createdAt}。 桌面「同步空间」改叫「云端工作区」，同一份数据。

② 默认工作区 quill

server 启动时确保存在：注册表条目（default: true，不可删）+ vault/quill/ 目录。web 未选择过工作区时进它。

③ 一次性迁移

启动时 vault 根下不属于任何已注册工作区的散文件 / 目录，整体移入 quill/ 并打日志。你服务器上的 1.json、2.json、h.md 会进默认工作区，web 上不丢内容。

RemoteVault 增加 rootPath（工作区目录前缀），list/read/write/move 全部自动加前缀， web 与桌面远程模式共用；vault API 本身不动。桌面端本地文件夹同步（上一期功能）行为不变， 只是绑定对象的叫法从「同步空间」变成「云端工作区」。
